# 起動フロー・基本操作

[overview](./overview.md) で定義した機能の、具体的な起動シーケンスとUI操作をまとめる。

## 起動時の更新シーケンス

アプリ起動直後、以下を順番に行う。**すべて完了するまで操作ボタンは表示しない。**

1. **BDS本体の更新確認**
   - Microsoftのサーバーに問い合わせ、統合版BDSの最新バージョンが出ているか確認する
   - バージョンが上がっていれば、BDS本体を更新する
   - 初回ダウンロード前にMinecraft EULAとプライバシーポリシーへの同意を求める
   - 公式download links APIの`serverBedrockWindows`を使用する

2. **専用ワールドデータの取得**
   - マイクラ人狼専用ワールドのデータは、公式Linuxサーバー側に置いてある
   - ローカルにワールドが無ければ、そこから取得する
   - (ローカルに既にある場合は取得しない → 差分更新するかは別途検討)

3. **アドオンの更新確認・適用**
   - 各アドオン (game-manager, vanillapack, additional-roles-1, kairo, kairo-database など) は GitHub Releases で配布される
   - 専用APIを使って更新情報を取得し、更新があれば適用する
   - Kairo RegistryのZIPからBP/RPをBDSへ同期し、manifest UUID/versionをworld pack JSONへ反映する

上記1〜3が完了して初めて、メイン画面のボタン群が出現する。

## メイン画面

更新シーケンス完了後、以下の2つのボタンを表示する。

### 「マイクラ人狼を起動」ボタン

アドオンには2種類ある。

- **必須アドオン**: game-manager, vanillapack, kairo, kairo-database など。常に導入する
- **任意アドオン**: additional-roles-1 など。ワールドに適用するかどうかを選べる

ボタンを押すと以下の流れになる。

1. 任意アドオンの選択UIを表示する
   - 前回選択した内容をデフォルト(初期チェック状態)として復元する
   - ユーザーはその場で選択を変更できる
2. 選択が確定したら、更新済みの専用ワールドに「必須アドオン + 選択された任意アドオン」を適用したうえで BDS を起動する
3. 起動後、接続先の IPアドレスとポートを画面に表示する

### 「別のワールドで遊ぶ」ボタン

押すとワールド一覧画面に遷移する。

- 過去に遊んだワールドなどを一覧表示する
- 右上に「新規作成」ボタンを配置する
  - 新規ワールドを作成してBDSを起動できる
  - `.mcworld` または `.zip` (非圧縮のフォルダ・ファイルでも可) のワールドデータをドラッグ&ドロップ(投げ込む)して、そのまま起動することもできる

## 制約

- 同時に起動できるBDSサーバーは **1つのみ**。
- 公開操作ではWindows Firewallを許可し、UPnP対応ルーターへUDP 19132のポート転送を設定する。
- UPnP非対応、二重ルーター、CGNAT環境では中継サーバーまたは手動設定が必要。
